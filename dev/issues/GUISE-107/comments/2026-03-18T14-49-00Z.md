A provisional `logback.xml` has been added to `cli/src/main/resources/` during work on [GUISE-230]. It pins `software.amazon.awssdk`, `org.apache.http`, and `io.netty` loggers to INFO so that `--debug` shows application debug messages without being overwhelmed by AWS SDK noise. This duplicates the base configuration from `globalmentor-application` with the logger pins added.

This is a short-term measure. Completing this ticket should evaluate whether the pins belong in the shared `globalmentor-application` `logback.xml` instead (eliminating duplication), and whether `--trace` should lift the pins to allow full third-party debug output when needed.
